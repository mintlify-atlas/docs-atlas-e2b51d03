## Overview

This guide will walk you through the basics of using VirtualizorJS to manage your Virtualizor VPS instances. You'll learn how to create a client, list VPS instances, and perform common operations.

<Steps>
  <Step title="Install VirtualizorJS">
    First, install VirtualizorJS using your preferred package manager:

    <CodeGroup>

    ```bash npm
    npm install virtualizorjs
    ```

    ```bash yarn
    yarn add virtualizorjs
    ```

    ```bash pnpm
    pnpm add virtualizorjs
    ```

    ```bash bun
    bun add virtualizorjs
    ```

    </CodeGroup>

    See the [Installation](/installation) guide for more details.
  </Step>

  <Step title="Create a Client">
    Import and create a Virtualizor client with your API credentials:

    ```typescript
    import { createVirtualizorClient } from 'virtualizorjs';

    const client = createVirtualizorClient({
      host: 'virtualizor.example.com',
      apiKey: 'your-api-key',
      apiPass: 'your-api-pass',
      // Optional configuration:
      port: 4085,           // Default: 4085 for HTTPS, 4084 for HTTP
      https: true,          // Default: true
      rejectUnauthorized: false,  // Default: false (allows self-signed certs)
      timeout: 30000,       // Default: 30000ms (30 seconds)
    });
    ```

    <Note>
      **Self-Signed Certificates**: Virtualizor typically uses self-signed SSL certificates. The SDK handles this by default with `rejectUnauthorized: false`. If you need to enforce certificate validation, set `rejectUnauthorized: true`.
    </Note>

    <Tip>
      Store your credentials in environment variables for security:
      ```typescript
      const client = createVirtualizorClient({
        host: process.env.VIRTUALIZOR_HOST!,
        apiKey: process.env.VIRTUALIZOR_API_KEY!,
        apiPass: process.env.VIRTUALIZOR_API_PASS!,
      });
      ```
    </Tip>
  </Step>

  <Step title="List VPS Instances">
    Retrieve all VPS instances from your Virtualizor server:

    ```typescript
    import { createVirtualizorClient, VirtualizorApiError } from 'virtualizorjs';

    const client = createVirtualizorClient({
      host: 'your-server.example.com',
      apiKey: 'your-api-key',
      apiPass: 'your-api-pass',
    });

    async function main() {
      try {
        const vpsList = await client.vps.list();

        console.log('VPS Instances:');
        for (const [vpsid, vps] of Object.entries(vpsList)) {
          console.log(`  ${vpsid}: ${vps.hostname}`);
          console.log(`    OS: ${vps.os_name}`);
          console.log(`    Status: ${vps.status}`);
        }
      } catch (error) {
        if (error instanceof VirtualizorApiError) {
          console.error(`API Error [${error.code}]: ${error.message}`);
        } else {
          console.error('Unexpected error:', error);
        }
      }
    }

    main();
    ```

    The `client.vps.list()` method returns a record where keys are VPS IDs and values are VPS objects.
  </Step>

  <Step title="Perform VPS Operations">
    VirtualizorJS supports all common VPS operations. Most operations are **asynchronous** and return a task ID that you can poll for completion.

    ### Start a VPS

    ```typescript
    // Start a VPS
    const result = await client.vps.start('123');
    console.log(`Task ID: ${result.taskid}`);
    ```

    ### Stop a VPS

    ```typescript
    // Stop a VPS (graceful shutdown)
    const result = await client.vps.stop('123');

    // Or power off immediately
    const poweroffResult = await client.vps.poweroff('123');
    ```

    ### Restart a VPS

    ```typescript
    const result = await client.vps.restart('123');
    ```

    ### Get VPS Details

    ```typescript
    const vps = await client.vps.get('123');
    console.log(`Hostname: ${vps.hostname}`);
    console.log(`OS: ${vps.os_name}`);
    console.log(`Status: ${vps.status}`);
    ```

    <Note>
      Most VPS operations like `start()`, `stop()`, `restart()`, `create()`, `rebuild()`, etc. are asynchronous and return an `AsyncTaskResult` with a `taskid`. Use the tasks API to poll for completion.
    </Note>
  </Step>

  <Step title="Poll for Task Completion">
    Many Virtualizor operations are asynchronous. Use the `client.tasks.wait()` method to poll until an operation completes:

    ```typescript
    import { createVirtualizorClient, VirtualizorApiError } from 'virtualizorjs';

    const client = createVirtualizorClient({
      host: 'your-server.example.com',
      apiKey: 'your-api-key',
      apiPass: 'your-api-pass',
    });

    async function main() {
      try {
        console.log('Starting VPS...');
        const result = await client.vps.start('123');

        const taskId = result.taskid;
        if (!taskId) {
          console.log('No task ID returned');
          return;
        }

        console.log(`Task ID: ${taskId}`);
        console.log('Waiting for task completion...');

        // Poll with custom options
        const taskResult = await client.tasks.wait(taskId, {
          pollIntervalMs: 2000,  // Check every 2 seconds (default)
          timeoutMs: 60000,      // Timeout after 60 seconds
        });

        if (taskResult.status === '1' || taskResult.status === 'done') {
          console.log('VPS started successfully!');
        } else {
          console.log('Task completed with status:', taskResult.status);
        }
      } catch (error) {
        if (error instanceof VirtualizorApiError) {
          console.error(`API Error [${error.code}]: ${error.message}`);
        } else if (error instanceof Error && error.message.includes('timeout')) {
          console.error('Task polling timed out');
        } else {
          console.error('Unexpected error:', error);
        }
      }
    }

    main();
    ```

    ### Polling Options

    The `wait()` method accepts optional configuration:

    | Option | Type | Default | Description |
    |--------|------|---------|-------------|
    | `pollIntervalMs` | `number` | `2000` | How often to check task status (in milliseconds) |
    | `timeoutMs` | `number` | `120000` | Maximum time to wait before timing out (in milliseconds) |

    <Tip>
      For long-running operations like VPS creation or migration, increase the `timeoutMs` value:
      ```typescript
      const task = await client.tasks.wait(taskId, {
        pollIntervalMs: 5000,   // Check every 5 seconds
        timeoutMs: 300000,      // Wait up to 5 minutes
      });
      ```
    </Tip>
  </Step>

  <Step title="Error Handling">
    All SDK errors are instances of `VirtualizorApiError`. Always catch and handle errors appropriately:

    ```typescript
    import { VirtualizorApiError } from 'virtualizorjs';

    try {
      await client.vps.start('invalid-id');
    } catch (err) {
      if (err instanceof VirtualizorApiError) {
        console.error(`API Error [${err.code}]: ${err.message}`);
        // Handle API-specific errors
      } else {
        console.error('Unexpected error:', err);
        // Handle other errors
      }
    }
    ```

    The `VirtualizorApiError` class includes:
    - `message`: Human-readable error description
    - `code`: Virtualizor API error code
    - Additional context from the API response
  </Step>
</Steps>

## Complete Example

Here's a complete example that creates a VPS, waits for creation to complete, and then starts it:

```typescript
import { createVirtualizorClient, VirtualizorApiError } from 'virtualizorjs';

const client = createVirtualizorClient({
  host: 'your-server.example.com',
  apiKey: 'your-api-key',
  apiPass: 'your-api-pass',
});

async function createAndStartVPS() {
  try {
    // Create a VPS
    console.log('Creating VPS...');
    const createResult = await client.vps.create({
      hostname: 'my-vps.example.com',
      rootpass: 'SecurePassword123!',
      osid: 1,  // OS template ID
    });

    // Wait for creation to complete
    console.log(`VPS creation task initiated: ${createResult.taskid}`);
    const createTask = await client.tasks.wait(createResult.taskid!, {
      pollIntervalMs: 5000,
      timeoutMs: 300000,  // 5 minutes
    });

    console.log('VPS created successfully!');

    // Get the new VPS ID from the task result
    // (You'll need to parse this from the task response)
    const newVpsId = '123';  // Replace with actual VPS ID

    // Start the VPS
    console.log('Starting VPS...');
    const startResult = await client.vps.start(newVpsId);
    await client.tasks.wait(startResult.taskid!);

    console.log('VPS is now running!');

    // Get VPS details
    const vps = await client.vps.get(newVpsId);
    console.log(`Hostname: ${vps.hostname}`);
    console.log(`Status: ${vps.status}`);
  } catch (error) {
    if (error instanceof VirtualizorApiError) {
      console.error(`API Error [${error.code}]: ${error.message}`);
    } else {
      console.error('Unexpected error:', error);
    }
  }
}

createAndStartVPS();
```
